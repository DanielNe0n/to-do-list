API Documentation

Introduction:

Our website provides an API (Application Programming Interface) that allows you to interact with your tasks programmatically. This API allows you to perform various operations such as retrieving all your tasks, creating new tasks, editing existing tasks, and deleting tasks.

Authentication:

To access the API, you need to authenticate using your user account credentials. You can obtain an API token by logging into your account on our website. This token will be required for all API requests as a means of authentication.


Token:
{{ token }}

API Endpoints:

Retrieve all tasks:
Endpoint: /api/v1/task/
Method: GET
Description: This endpoint allows you to retrieve all tasks associated with your account. The response will be a JSON array containing information about each task, such as task title, description, due date, and status.

Create a new task:
Endpoint: /api/v1/task/
Method: POST
Description: This endpoint (POST method) allows you to create a new task. You need to provide the task details in the request body in JSON format, including the task title, description, and status. The API will return a JSON object containing the newly created task's details, including its unique ID.

Edit an existing task:
Endpoint: /api/v1/task/<task_id>/
Method: PATCH
Description: This endpoint allows you to edit an existing task by providing its unique ID in the URL. You need to provide the updated task details in the request body in JSON format, including the task title, description, due date, and status. The API will return a JSON object containing the updated task's details.

Delete a task:
Endpoint: /api/v1/task/<task_id>/
Method: DELETE
Description: This endpoint allows you to delete a task by providing its unique ID in the URL. The API will return a JSON object confirming the deletion of the task.

Response Format:

The API responses are in JSON format and contain relevant information about the tasks, such as task title, description, due date, and status. If an error occurs, the API will return an appropriate error message in the JSON response, indicating the reason for the error.
